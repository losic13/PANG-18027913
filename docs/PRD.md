# PANG 게임 - 미션 1 PRD

## 배경

Pang(팡)은 Mitchell사가 출시한 아케이드 게임(북미명 Buster Bros.)을 원형으로 한다. 화면 위에서 통통 튀는 풍선(공)을 무기로 맞혀 계속 더 작은 크기로 쪼개다가, 가장 작은 공까지 모두 없애면 스테이지를 클리어하는 방식이 핵심 규칙이다. 이번 미션 1은 이 원형 게임의 기본 규칙만 담아 최소 플레이 가능 버전(MVP)을 구현하는 것을 목표로 한다.

## 목표

브라우저에서 실행되는 웹 버전 Pang 게임의 1단계(미션 1)를 구현한다. 복잡한 파워업이나 다단계 스테이지 없이, 공 하나를 완전히 없애면 성공하는 가장 단순한 형태로 핵심 루프(조작 → 발사 → 충돌 → 분할 → 승패 판정)를 완성하는 데 집중한다.

## 핵심 규칙

- **캐릭터 조작**: 캐릭터(PLAYER)는 화면 하단에 위치하며 좌우 방향키로만 이동할 수 있다. 점프는 없다.
- **Wire 발사**: 스페이스 키를 누르면 PLAYER 위치에서 수직 위 방향으로만 Wire를 쏘아 올린다. Wire는 벽(Block)을 뚫지 못한다.
- **Bubble의 움직임**: 큰 Bubble 1개가 화면에 등장해 중력의 영향을 받으며 통통 튀어 다닌다. 바닥/Block/벽에 부딪히면 반대 방향으로 튕기며, 튕길 때마다 직전과 같은 높이까지 다시 올라간다. 화면 밖으로는 나갈 수 없다.
- **분할 규칙**: Wire가 Bubble에 닿으면 그 Bubble은 더 작은 크기 2개로 분할된다. 가장 작은 크기의 Bubble은 Wire에 맞으면 사라진다.
- **Block**: 부서지지 않는 고정 지형이며, Bubble은 Block 위로 튕겨 오를 수 있다.
- **충돌 판정**: PLAYER가 Bubble에 닿으면 목숨을 하나 잃는다. 보유한 목숨을 모두 잃으면 게임이 실패로 종료된다.

## 승패 조건

| 조건 | 결과 |
|---|---|
| 화면 위의 Bubble을 모두 없앰 | 성공 (게임 종료) |
| PLAYER가 보유한 목숨을 모두 소진 | 실패 (게임 종료) |
| 제한 시간 99초 초과 | 실패 (게임 종료) |

## 기술 기준

- 게임 루프는 FPS 30을 기준으로 고정하고, 속도감 조절이 필요하면 FPS 대신 각 오브젝트의 speed 값을 조정한다.
- 기존 저장소에 확정된 [기술 스택](../CLAUDE.md)(Vite + React + TypeScript)을 그대로 사용한다.
