# PLAN_MISSION2.md

미션 2(BLOCK 스테이지 도입) 구현을 위한 단계별(Phase) 개발 계획이다. [PLAN.md](PLAN.md)(미션 1)와 동일한 원칙을 따른다: 각 Phase는 **직접 실행하고 눈으로 확인할 수 있는 동작하는 결과물**을 목표로 하며, 이전 Phase 위에 기능을 하나씩만 얹는다. Phase가 끝나면 고객이 직접 실행해 해당 기능만 집중적으로 테스트하고 피드백을 준 뒤 다음 Phase로 넘어간다.

참고 문서: [MISSION_ROADMAP.md](MISSION_ROADMAP.md), [게임 룰 상세](docs/FEATURES/game_rule.md), [미션1](docs/FEATURES/mission1.md)

## 미션 2 개요

- 미션 1의 핵심 루프(조작 → Wire 발사 → 충돌 → 분할 → 승패 판정)는 그대로 유지한다.
- 좌우 벽/바닥 외에 **고정 지형 BLOCK**을 스테이지에 추가한다.
- PLAYER는 바닥 평면에서만 이동하며 BLOCK과는 상호작용하지 않는다(BLOCK 아래를 그대로 지나다닌다). BLOCK은 오직 Bubble의 발판, Wire의 장애물로만 기능한다.
- Bubble 개수는 미션 1과 동일하게 1개만 유지한다(다중 Bubble은 미션 3 범위).

---

## Phase 1. BLOCK 정적 배치 노출

**목표**
- 스테이지 내 특정 위치에 BLOCK 1개를 고정된 사각형으로 렌더링한다.
- 아직 Bubble/Wire/PLAYER와의 충돌 로직은 없음(시각적 배치만 검증).

**고객이 확인할 포인트**
- BLOCK의 위치와 크기가 스테이지 레이아웃상 자연스러운지
- 기존 미션 1 플레이(이동/발사/충돌)가 BLOCK 존재와 무관하게 그대로 동작하는지(아직은 그냥 통과되어야 함)

---

## Phase 2. WIRE-BLOCK 충돌

**목표**
- Wire가 BLOCK에 닿으면 Bubble에는 영향을 주지 않고 그 자리에서 Wire만 소멸한다.

**고객이 확인할 포인트**
- Wire가 BLOCK 아래쪽에서 발사되었을 때 정확히 BLOCK 하단에서 멈춰 사라지는지
- BLOCK을 피해서 발사한 Wire는 기존과 동일하게 화면 상단까지 정상 도달/소멸하는지

---

## Phase 3. BUBBLE-BLOCK 상단 반사(발판 역할)

**목표**
- Bubble이 BLOCK 윗면에 떨어지면 바닥에 닿았을 때와 동일한 규칙으로 튕겨 오른다(직전과 같은 높이까지 재상승).

**고객이 확인할 포인트**
- BLOCK 위에서 튕기는 높이가 바닥에서 튕길 때와 동일하게 일정한지
- BLOCK 위에 착지하는 순간의 판정이 자연스러운지(관통하거나 파묻히지 않는지)

---

## Phase 4. BUBBLE-BLOCK 좌우/하단 충돌

**목표**
- Bubble이 BLOCK의 옆면이나 아랫면에 부딪히는 경우에도 벽에 부딪힌 것과 동일하게 진행 방향이 반전된다.

**고객이 확인할 포인트**
- BLOCK 옆면에 부딪힌 Bubble이 자연스럽게 반대 방향으로 튕겨나가는지
- BLOCK 아래쪽에서 위로 튀어 오르다 BLOCK 밑면에 부딪히는 경우에도 위치 튕김(끼임 없이) 처리가 정확한지

---

## Phase 5. 다중 BLOCK 배치 및 스테이지 레이아웃 확정

**목표**
- BLOCK을 2~3개 이상 배치해 실제 스테이지 레이아웃을 구성한다.
- 배치 데이터를 상수/설정 형태로 분리해 이후 미션(다중 스테이지)에서 재사용할 수 있는 형태로 정리한다.

**고객이 확인할 포인트**
- 여러 BLOCK이 동시에 존재할 때도 Bubble/Wire 충돌 판정이 빠짐없이 정확하게 동작하는지
- 실제 플레이 시 BLOCK 배치가 난이도와 재미 측면에서 자연스러운지(막힘/사각지대 없는지)
- 미션 1 대비 전체 플레이 흐름(승패 판정 포함)이 여전히 정상 동작하는지

---

미션 2가 완료되면 미션 3(여러 Bubble 동시 등장) 착수 여부를 고객과 논의한다.
